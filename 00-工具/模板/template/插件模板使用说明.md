# 插件文档模板使用说明

## 📋 快速开始

### 第一步：安装 Templater 插件

1. 在 Obsidian 中打开 **设置**
2. 进入 **第三方插件** → **浏览**
3. 搜索并安装 **"Templater"** 插件
4. 启用插件

### 第二步：配置模板文件夹

1. 在 Obsidian 设置中找到 **Templater** 插件设置
2. 设置 **模板文件夹位置** 为当前 `template` 目录
3. 启用 **"启动时自动运行 Templater"**

### 第三步：创建新插件文档

1. 使用快捷键 `Ctrl/Cmd + P` 打开命令面板
2. 输入 **"Templater"**，选择 **"插入模板"**
3. 选择 **"插件文档模板"**
4. 按提示填写以下信息：
   - 插件名称（英文）
   - 插件类型（Obsidian/Chrome/VSCode/NPM/Python）
   - 功能分类（可多选）
   - 配置难度（⭐-⭐⭐⭐⭐⭐）
   - 价格类型（免费/付费/免费+付费/开源）

## 📊 YAML 字段详解

### 基础标识字段

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| `uuid` | string | ✅ | 全局唯一标识符 | `20241223-插件-001` |
| `type` | string | ✅ | 固定为 Tool | `Tool` |
| `plugin_type` | string | ✅ | 插件平台类型 | `obsidian`, `chrome`, `vscode` |
| `plugin_name` | string | ✅ | 插件英文名称 | `Gemini CLI` |
| `plugin_category` | array | ✅ | 插件功能分类 | `[AI, 工作流]` |
| `tags` | array | ❌ | 自定义标签 | `[Google, Gemini]` |
| `status` | string | ✅ | 文档成熟度 | `Seedling`, `Budding`, `Evergreen` |
| `aliases` | array | ❌ | 检索别名 | `[Gemini CLI, Gemini命令行]` |

### 插件属性字段

| 字段 | 类型 | 说明 | 可选值 |
|------|------|------|--------|
| `author` | string | 作者/开发团队 | - |
| `official_site` | string | 官方网站 URL | - |
| `repository` | string | 代码仓库 URL | - |
| `pricing` | string | 价格类型 | `免费`, `付费`, `免费+付费`, `开源` |
| `language` | string | 支持语言 | `中文`, `英文`, `多语言` |
| `difficulty` | string | 配置难度 | `⭐` 到 `⭐⭐⭐⭐⭐` |
| `time_cost` | string | 学习时间成本 | `5分钟`, `15分钟`, `30分钟`, `1小时+` |

### 质量控制字段

| 字段 | 类型 | 说明 | 可选值 |
|------|------|------|--------|
| `status` | string | **【关键字段】**文档成熟度 | `Seedling` (🌱 草稿)<br>`Budding` (🌿 修改中)<br>`Evergreen` (🌲 定稿) |
| `content_completeness` | string | 内容完整度 | `草稿`, `完整`, `详细` |
| `review_status` | string | 审核状态 | `待审核`, `已审核`, `已验证` |
| `last_tested` | date | 最后测试日期 | ISO 8601 格式 |
| `created` | date | 创建日期 | ISO 8601 格式 |
| `updated` | date | 更新日期 | ISO 8601 格式 |

### 关联信息字段

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `related_plugins` | array | 相关插件 | `[Obsidian Terminal, Templater]` |
| `related_concepts` | array | 相关概念 | `[RAG, AI知识库]` |
| `use_cases` | array | 使用场景 | `[AI辅助写作, 素材检索]` |
| `source` | string | 来源作者/网站 | - |
| `source_url` | string | 来源 URL | - |

## 🎯 文档质量控制流程

### 阶段 1: 草稿 (Seedling 🌱)
**适用场景**: 初步收集插件信息，内容不完整

**必需字段**:
- ✅ 基础标识字段（uuid, type, plugin_name, plugin_type）
- ✅ status: `Seedling`
- ✅ created, updated
- ❌ 其他字段可选

**内容要求**:
- 基本的插件介绍
- 简单的安装步骤
- 至少 1 个使用场景

---

### 阶段 2: 修改中 (Budding 🌿)
**适用场景**: 正在补充细节，内容较完整

**必需字段**:
- ✅ 所有基础标识字段
- ✅ 插件属性字段（author, pricing, difficulty, time_cost）
- ✅ status: `Budding`
- ✅ last_tested
- ✅ related_plugins (至少 1 个)

**内容要求**:
- 完整的安装步骤
- 至少 3 个使用场景
- 优缺点分析
- 注意事项
- 相关资源链接

---

### 阶段 3: 定稿 (Evergreen 🌲)
**适用场景**: 内容成熟，经过验证，可以正式引用

**必需字段**:
- ✅ 所有字段完整填写
- ✅ status: `Evergreen`
- ✅ content_completeness: `详细` 或 `完整`
- ✅ review_status: `已验证`
- ✅ 所有关联信息字段已填写

**内容要求**:
- 安装步骤经过测试验证
- 使用场景有实际案例
- 包含高级技巧
- 有完整的优缺点分析
- 相关资源链接有效
- Dataview 查询可用

## 🔍 使用 Dataview 查询插件

### 查询所有 AI 类插件

\`\`\`dataview
TABLE plugin_name, pricing, difficulty, status
FROM "插件"
WHERE contains(plugin_category, "AI")
SORT difficulty ASC
\`\`\`

### 查询所有免费插件

\`\`\`dataview
TABLE plugin_name, plugin_type, difficulty
FROM "插件"
WHERE pricing = "免费" AND status = "Evergreen"
\`\`\`

### 查询某个难度的插件

\`\`\`dataview
TABLE plugin_name, time_cost, plugin_category
FROM "插件"
WHERE difficulty = "⭐⭐⭐"
SORT updated DESC
\`\`\`

### 查询最近更新的插件

\`\`\`dataview
LIST
FROM "插件"
WHERE updated >= date(today) - dur(30 days)
SORT updated DESC
\`\`\`

## 📝 最佳实践

### 1. 文件命名规范

**推荐格式**:
\`\`\`
插件{编号} {插件名称}.md
\`\`\`

**示例**:
- ✅ `插件1 Google Gemini CLI.md`
- ✅ `插件2 Excalidraw.md`
- ✅ `插件3 Dataview.md`

### 2. UUID 生成规则

\`\`\`
{YYYYMMDD}-插件-{序号}
\`\`\`

**示例**: `20241223-插件-001`

### 3. 定期维护

- **每周**: 检查 Seedling 状态的插件，推进到 Budding
- **每月**: 测试已记录的插件，更新 last_tested 日期
- **每季度**: review 所有 Evergreen 插件，更新内容

### 4. 内容质量控制

**判断标准**:
- 是否真实测试过该插件？
- 安装步骤是否可以复现？
- 是否有实际的使用案例？
- 优缺点分析是否客观？
- 相关资源链接是否有效？

## 🚀 高级技巧

### 技巧 1: 使用 MetaEdit 批量更新

如果安装了 MetaEdit 插件，可以批量修改元数据：

\`\`\`dataview
TABLE plugin_name, status
FROM "插件"
WHERE status = "Seedling"
\`\`\`

然后使用 MetaEdit 批量修改状态。

### 技巧 2: 创建插件索引页

在 `插件/` 目录下创建 `_索引.md`:

\`\`\`markdown
# 插件索引

## 按类型分类

\`\`\`dataview
TABLE plugin_name, pricing
FROM "插件"
GROUP BY plugin_type
\`\`\`

## 按难度分类

\`\`\`dataview
TABLE plugin_name, time_cost
FROM "插件"
WHERE status = "Evergreen"
GROUP BY difficulty
\`\`\`
\`\`\`

### 技巧 3: 插件推荐系统

根据你已有的插件，自动推荐相关插件：

\`\`\`dataview
TABLE plugin_name, plugin_category
FROM "插件"
WHERE contains(related_plugins, [[当前插件]])
\`\`\`

## ⚠️ 常见问题

### Q: 为什么要如此复杂的结构？

**A**: 结构化的元数据可以实现：
- ✅ 精准的 AI 检索
- ✅ 自动化查询和分析
- ✅ 质量控制和维护
- ✅ 知识图谱构建

### Q: 哪些字段是必须的？

**A**:
- **最少必须**: `uuid`, `type`, `plugin_name`, `plugin_type`, `status`, `created`, `updated`
- **推荐完整**: 所有字段都填写，以达到 Evergreen 状态

### Q: 如何处理已有插件文档？

**A**:
1. 逐步迁移，优先迁移常用的插件
2. 可以使用批量操作工具添加 frontmatter
3. 从 Seedling 开始，逐步完善到 Evergreen

### Q: 可以自定义字段吗？

**A**: 可以！根据你的需求添加：
- `favorite`: true/false (收藏标记)
- `frequency`: 经常使用/偶尔使用/很少使用
- `rating`: 1-5 星评分
