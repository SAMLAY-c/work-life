# 智能知识工程模板使用指南

## 🎯 模板核心理念

这个模板基于**智能知识工程**原则设计，旨在将传统的"文科生笔记"转化为"工程师数据库"，让 AI (特别是 RAG 系统) 能够完美理解和检索你的知识。

## 📋 模板组件说明

### 1. YAML Frontmatter - 笔记的"身份证"

```yaml
---
uuid: 20241220-01      # 全局唯一标识符
type: Concept          # 笔记类型
tags: [AI, RAG]        # 分类标签
status: Evergreen      # 成熟度状态
aliases: [检索增强生成] # 检索别名
created: 2024-12-20T10:30:00  # 创建时间
updated: 2024-12-20T10:30:00  # 更新时间
related: []            # 相关笔记链接
source: ""             # 来源引用
---
```

#### 字段详解：

- **uuid**: 笔记的唯一身份证，文件名可能改变但 uuid 不变
- **type**: 笔记分类，支持以下类型：
  - `Concept` (概念)
  - `Project` (项目)
  - `Meeting` (会议)
  - `Person` (人物)
  - `Technique` (技术)
  - `Framework` (框架)
  - `CaseStudy` (案例)
  - `Tool` (工具)

- **status**: **【质量控制关键字段】**
  - `Seedling` (🌱 草稿): 初步想法，内容不完整
  - `Budding` (🌿 修改中): 正在迭代，内容较完整
  - `Evergreen` (🌲 定稿): 内容成熟，可用于正式检索

- **aliases**: **【检索增强关键字段】** 同一概念的不同表述方式

## 🔧 使用步骤

### 第一步：安装 Templater 插件

1. 在 Obsidian 中打开设置
2. 进入"第三方插件" → "浏览"
3. 搜索并安装 "Templater" 插件
4. 启用插件

### 第二步：配置模板文件夹

1. 在 Obsidian 设置中找到 Templater 插件设置
2. 设置模板文件夹为当前 `template` 目录
3. 启用"启动时自动运行 Templater"

### 第三步：创建新笔记

1. 使用 `Ctrl/Cmd + N` 新建笔记
2. 在 Templater 菜单中选择"智能知识工程模板"
3. 按照提示填写各个字段
4. 使用 `Tab` 键在光标位置之间切换

## 📝 使用最佳实践

### 1. 原子化笔记原则

**❌ 错误示例：**
```
# AI学习心得
今天学习了Transformer、RAG、Midjourney...
```

**✅ 正确示例：**
- `Transformer架构.md` - 只讲Transformer
- `RAG配置指南.md` - 只讲RAG配置
- `Midjourney提示词技巧.md` - 只讲提示词

### 2. 双向链接使用技巧

**基础链接：**
- 这个概念扩展了 [[机器学习基础]]
- 这个技术解决了 [[数据处理难题]]

**关系化链接：**
- 这个理论 **反驳了** [[传统方法]] 中的观点
- 这个案例 **证明了** [[相关理论]] 的有效性
- 这个工具 **基于** [[底层技术]] 实现

### 3. 质量控制流程

1. **草稿阶段 (Seedling)**: 随时记录想法，不追求完整
2. **迭代阶段 (Budding)**: 补充细节，建立链接，完善内容
3. **定稿阶段 (Evergreen)**: 内容完整，逻辑清晰，可以引用

### 4. AI 检索优化

当使用 AI 检索时，可以利用元数据进行精准过滤：

```
"请总结所有 status: Evergreen 且 type: Concept 的笔记"
"查找所有包含 RAG 标签的技术文档"
"显示所有2024年创建的定稿笔记"
```

## 🚀 高级技巧

### 1. 批量更新状态
使用 Obsidian 搜索功能批量查找和更新笔记状态：

```
path:"03-产品需求" status:Seedling
```

### 2. 知识图谱构建
定期检查笔记的链接关系：
- 哪些笔记被频繁引用？
- 哪些概念缺少相关链接？
- 是否存在孤立的笔记？

### 3. 内容维护
- 定期 review Seedling 状态的笔记
- 及时更新 `updated` 字段
- 保持 alias 的实用性

## 🔗 与现有知识库的集成

这个模板设计为与你的现有知识结构无缝集成：

- `prd/` 目录 → 使用 `Project` 类型
- `advise/` 目录 → 使用 `Framework` 类型
- `增长黑客/` 目录 → 使用 `Concept` 类型
- 具体分析文档 → 使用 `CaseStudy` 类型

## ⚠️ 常见问题

**Q: 为什么需要如此复杂的结构？**
A: 这是为了让 AI 能够进行精确的语义检索和推理，避免"幻觉"和信息混乱。

**Q: 可以简化某些字段吗？**
A: `uuid` 和 `status` 是核心质量控制字段，不建议简化。其他字段可以根据需要调整。

**Q: 如何处理已有笔记？**
A: 建议逐步迁移，优先迁移重要且常用的笔记。

---

记住：**"重思考轻PRD"** 的理念同样适用于知识管理。结构化的思考比单纯的内容积累更重要。
