# 任务全流程管理

> 从捕获到复盘的完整任务管理流程

---

## 🔄 任务全流程图

```
灵感捕获 → 任务拆解 → 排期设置 → 每日执行 → 定期复盘
    ↓         ↓         ↓         ↓         ↓
  灵感库    任务组    日期标记    看板视图   回顾调整
```

## 📥 步骤1：捕获与拆解

### 1.1 灵感捕获

**什么时候需要捕获？**
- 突然想到的想法
- 会议中记录的待办
- 看到文章后的启发
- 任何"以后可能要做"的事情

**捕获方式**：

#### 方式A：使用 QuickAdd 快速记录（推荐）

1. 按 `Cmd/Ctrl + P` 打开命令面板
2. 输入 "QuickAdd"
3. 选择"快速任务"宏
4. 输入内容，例如："学习 Rust 语言"
5. 任务会自动添加到 `灵感库/` 下的"待整理想法.md"

#### 方式B：手动记录到灵感库

直接在 `灵感库/` 文件夹下创建笔记或添加到现有笔记：

```markdown
## 待整理想法

- [ ] 学习 Rust 语言
- [ ] 尝试断糖饮食
- [ ] 给父母买个智能音箱
- [ ] 整理书架
```

### 1.2 从灵感到项目

**定期整理灵感库**（建议每周一次）

1. 打开 `灵感库/待整理想法.md`
2. 评估每个想法：
   - **不值得做** → 删除
   - **简单琐事** → 直接设置日期，变成日常任务

### 1.3 任务拆解原则

**MECE原则**（相互独立，完全覆盖）

❌ **错误示范**：
```markdown
### 准备考试
- [ ] 复习
- [ ] 做题
- [ ] 背单词
```

**问题**："复习"太大，不具体；"背单词"和"复习"可能有重叠

✅ **正确示范**：
```markdown
### 第一轮复习（2周）
- [ ] 整理第一章笔记 📅 2025-01-01
- [ ] 完成第一章习题 📅 2025-01-02
- [ ] 整理第二章笔记 📅 2025-01-03
- [ ] 完成第二章习题 📅 2025-01-04

### 错题本整理（每周）
- [ ] 整理本周错题 🔁 every week on Sunday

### 模拟考试（最后2周）
- [ ] 完成第一套模拟题 📅 2025-01-20 ⏫
- [ ] 完成第二套模拟题 📅 2025-01-25
- [ ] 完成第三套模拟题 📅 2025-01-30
```

**拆解技巧**：
1. **按时间拆**：第一周、第二周、...
2. **按模块拆**：模块A、模块B、...
3. **按步骤拆**：准备、执行、验收、复盘
4. **SMART原则**：具体、可衡量、可达成、相关性、有时限

## 📅 步骤2：排期

### 2.1 排期的核心原则

**原则1：精确到"天"，不精确到"时"**

❌ 不要这样：
```markdown
- [ ] 完成项目报告 📅 2025-01-15 14:00
```
**原因**：现实中总有突发情况，精确到小时容易被打乱

✅ 应该这样：
```markdown
- [ ] 完成项目报告 📅 2025-01-15
```
**优点**：当天完成即可，灵活性更高

---

**原则2：使用 Tasks 插件的日期类型**

Tasks 插件支持多种日期类型：

| 日期类型 | 语法 | 含义 | 使用场景 |
|---------|------|------|----------|
| **Due（截止日期）** | `📅 2025-01-01` | 必须在这天完成 | 有明确deadline的任务 |
| **Scheduled（计划日期）** | `⏳ 2025-01-01` | 计划在这天做 | 希望在这天完成，但可以延期 |
| **Start（开始日期）** | `🛫 2025-01-01` | 最早在这天开始 | 不能提前开始的任务 |

**推荐使用**：
- 有deadline → 用 `📅` (Due)
- 无deadline但希望安排 → 用 `⏳` (Scheduled)

## 🎯 步骤3：执行

### 3.1 构建"轻重缓急看板"

**创建文件**：`00-系统/任务看板.md`

这是你每天工作的核心界面，使用改良版四象限矩阵：

```markdown
# 轻重缓急看板

> 更新时间：`<% tp.date.now("YYYY-MM-DD HH:mm") %>`

## 🔴 紧急且重要（本周 + 截止今天）

**定义**：本周截止 + 今天必须完成 + 高优先级

```dataview
TASK
WHERE !completed
AND due <= date(today)
AND due >= date(today) - dur(7 days)
AND priority = "high"
SORT due ASC
GROUP BY due
```

---

## 🟡 紧急不重要（本周 + 截止今天）

**定义**：本周截止 + 今天必须完成 + 普通/低优先级

```dataview
TASK
WHERE !completed
AND due <= date(today)
AND due >= date(today) - dur(7 days)
AND (priority = "medium" OR priority = "low" OR priority = null)
SORT due ASC
GROUP BY due
```

---

## 🟢 不急但重要（本周 + 截止明天后）

**定义**：本周截止 + 明天及以后 + 高优先级

```dataview
TASK
WHERE !completed
AND due > date(today)
AND due <= date(today) + dur(7 days)
AND priority = "high"
SORT due ASC
GROUP BY due
```

---

## 🔵 不急不重要（本周 + 截止明天后）

**定义**：本周截止 + 明天及以后 + 普通/低优先级

```dataview
TASK
WHERE !completed
AND due > date(today)
AND due <= date(today) + dur(7 days)
AND (priority = "medium" OR priority = "low" OR priority = null)
SORT due ASC
GROUP BY due
```

### 3.2 每日工作流程

**早上（10分钟）**：
1. 打开 `00-系统/任务看板.md`
2. 查看"紧急且重要"区域
3. 选择1-3个任务作为今天的重点
4. 在任务描述后添加 `🎯` 标记：

```markdown
- [ ] 完成项目报告 📅 2025-01-15 ⏫ 🎯
```

**工作中（随时）**：
- 完成任务后，点击复选框标记为完成
- Tasks 插件会自动添加完成日期 ✅ 2025-01-15

**晚上（5分钟）**：
1. 回顾今天完成的任务
2. 将未完成但应该继续的任务标记 `🎯`
3. 如果某任务今天无法完成，考虑调整due日期

---

### 3.3 任务执行技巧

**技巧1：番茄工作法**

对于需要深度工作的任务：
- 设置25分钟专注 + 5分钟休息
- 在任务中记录番茄数：

```markdown
- [ ] 完成项目报告 📅 2025-01-15 ⏫ 🎯
  - 番茄1：完成大纲
  - 番茄2：完成第一章
  - 番茄3：完成第二章
```

---

**技巧2：任务分解**

如果一个任务太大，无法一次完成：

**原任务**：
```markdown
- [ ] 完成项目报告 📅 2025-01-15 ⏫
```

**分解为**：
```markdown
- [x] 完成项目报告大纲 ✅ 2025-01-14
- [ ] 完成第一章内容 📅 2025-01-15
- [ ] 完成第二章内容 📅 2025-01-15
- [ ] 校对和格式调整 📅 2025-01-15
```

---

**技巧3：任务阻塞**

如果任务被阻塞（等待他人回复、资源未到位）：

```markdown
- [ ] 等待设计师提供素材（阻塞中）📅 2025-01-15 ⏸️
```

添加 `⏸️` 标记，这样在 Dataview 查询中可以过滤掉

### 4.2 延期任务处理

当任务延期时，使用 **EEDM模型** 诊断：

**创建文件**：`00-系统/任务回顾.md`

```markdown
# 任务回顾

## 📌 延期任务列表

```dataview
TASK
WHERE !completed
AND due < date(today)
AND !contains(text, "⏸️")
GROUP BY file.link
SORT due ASC
```

## 🔍 EEDM 诊断模型

对于每个延期任务，问自己以下问题：

### E - Expectations（预期不合理？）

**症状**：任务太大、时间太紧、资源不足

**解决方案**：
- 拆分任务
- 延长截止日期
- 降低完成标准（从100分做到80分）

---

### E - Energy（精力不足？）

**症状**：状态不好、疲劳、缺乏动力

**解决方案**：
- 修改项目 `status` 为 `已暂停`
- 休息调整后再改回 `进行中`
- 将任务移到精力充沛的时间段

---

### D - Difficulty（难度过高？）

**症状**：技能不够、任务超出能力范围

**解决方案**：
- 学习必要技能后再做
- 寻求他人帮助
- 降低任务难度

---

### M - Motivation（动力不足？）

**症状**：不想做、拖延、觉得不重要

**解决方案**：
- **EEDM终极判断**：如果真的不重要，**删除任务**
- 重新评估任务的价值
- 找到任务的"为什么"

---

### 4.3 项目暂停与重启

**如何暂停项目**：

1. 打开项目笔记
2. 修改 `status` 属性：

```yaml
---
status: 已暂停
pause_reason: 精力不足，需要先完成其他项目
pause_date: 2025-01-15
restart_plan: 2025-02-01
---
```

3. 该项目的所有任务会自动从看板消失（因为 Dataview 查询会过滤 `status: 已暂停`）

---

**如何重启项目**：

1. 打开已暂停的项目笔记
2. 修改 `status` 为 `进行中`
3. 重新评估任务的 due 日期
4. 任务会自动重新出现在看板

---

## 🎯 完整流程示例

**场景**：你决定"90天减脂"

**Day 1（捕获）**：
- 在灵感库记录："尝试断糖减肥"

**Day 7（整理）**：
- 将灵感转化为项目：`01 个人/2025年减脂计划.md`
- 拆解任务组：饮食、运动、监测
- 设置属性：`status: 规划`

**Day 8（排期）**：
- 给前10个任务设置 due 日期
- 修改 `status: 计划`

**Day 15（开始执行）**：
- 修改 `status: 进行中`
- 任务自动出现在"任务看板"
- 每天查看"紧急且重要"区域，执行任务

**Day 22（第一周复盘）**：
- 完成5个任务，延期2个
- 延期原因：任务难度预估不足
- 调整后续任务的due日期

**Day 85（遇到问题）**：
- 平台期，体重不下降，动力不足
- 使用 EEDM 诊断：属于 Motivation 问题
- 决定暂停1周，调整心态

**Day 92（重启）**：
- 修改 `status: 进行中`
- 继续执行

**Day 180（完成）**：
- 完成所有任务
- 修改 `status: 已完成`
- 撰写项目总结
