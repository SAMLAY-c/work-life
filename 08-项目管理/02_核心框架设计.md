# 核心框架设计

> 本文档详细介绍任务管理系统的核心设计理念与文件夹结构

---

## 🎯 设计理念：关系系统 vs. 紧急重要矩阵

### ❌ 传统方式的问题

传统的"艾森豪威尔矩阵"（四象限法则）存在以下问题：

```
┌─────────────┬──────────────┬──────────────┐
│             │     重要      │    不重要     │
├─────────────┼──────────────┼──────────────┤
│     紧急    │  第一象限     │  第三象限     │
│   不紧急    │  第二象限     │  第四象限     │
└─────────────┴──────────────┴──────────────┘
```

**痛点**：
- ❌ "重要"和"紧急"很难界定（什么是"重要"？）
- ❌ 同一个任务可能在不同时间属于不同象限
- ❌ 容易陷入"紧急"陷阱，忽视长期"重要"任务
- ❌ 缺乏生活场景的关联性

---

### ✅ 关系系统（本教程推荐）

按照 **生活领域** 来分类，更符合人脑的认知模式：

**核心逻辑**：
- 我们的精力是按"生活角色"分配的
- 同一领域的任务可以互相参考和借鉴
- 避免"工作压倒生活"的问题

## 🏷️ 项目笔记的标准格式

每个"大目标/计划"都应该是一个独立的笔记，并包含以下要素：

### 1. 属性（Properties / YAML Frontmatter）

在笔记开头使用 **属性**（Obsidian的新功能）或 **YAML**：

```yaml
---
status: 进行中          # 状态：规划/计划/进行中/已暂停/已完成
type: project           # 类型：project/日常任务/习惯
priority: high          # 优先级：high/medium/low
created: 2025-01-01     # 创建日期
deadline: 2025-03-31    # 截止日期（可选）
estimated_tasks: 15     # 预估任务数
estimated_hours: 50     # 预估耗时（小时）
---
```

**状态说明**：

| 状态 | 说明 | 在看板中的行为 |
|------|------|----------------|
| **规划** | 还在思考阶段，任务尚未拆解 | 不出现在看板 |
| **计划** | 任务已拆解，等待开始 | 不出现在看板 |
| **进行中** | 正在执行 | 看板显示所有任务 |
| **已暂停** | 暂时搁置（根据EEDM诊断） | 看板不显示任务 |
| **已完成** | 项目完成 | 看板显示统计信息 |

---

### 2. 任务组的拆解原则

不要把一个大目标写成一行任务，而是拆解为 **"任务组"**：

#### ❌ 错误示范

```markdown
## 2025年减脂计划

- [ ] 减肥
```

**问题**：任务太大，无法执行，缺乏可操作性

---

#### ✅ 正确示范

```markdown
## 2025年减脂计划

### 饮食管理任务组
- [ ] 制定第一周的低糖食谱 📅 2025-01-01
- [ ] 清理冰箱中的不健康食品 📅 2025-01-01
- [ ] 购买一周的健康食材 📅 2025-01-02
- [ ] 准备5个工作日的健康午餐 📅 2025-01-03
- [ ] 学习阅读营养成分表 📅 2025-01-05

### 运动计划任务组
- [ ] 制定每周运动计划（3次有氧+2次力量） 📅 2025-01-01
- [ ] 购买运动装备（跑鞋、运动服、瑜伽垫） 📅 2025-01-05
- [ ] 第一次户外跑步30分钟 📅 2025-01-07 ⏫
- [ ] 制定室内替代方案（雨天） 📅 2025-01-10

### 监测与复盘任务组
- [ ] 记录初始体重和体脂率 📅 2025-01-01
- [ ] 设置每周固定称重时间（周一早上） 📅 2025-01-01
- [ ] 准备体重记录表格 📅 2025-01-02
- [ ] 第一次周复盘 📅 2025-01-08 ⏫
```

**任务组设计原则**：
1. **MECE原则**：相互独立，完全覆盖
2. **可执行性**：每个任务都是具体行动（不是"学习"而是"完成第一章"）
3. **时间顺序**：按照逻辑顺序排列
4. **优先级标记**：关键任务用 `⏫`（高优先级）标记

### 3. 双链的使用（超大规划）

当一个规划过于庞大时，可以使用 **双链** 来连接多个子规划：

#### 示例："健康生活"超大规划

**主笔记：`01 个人/健康生活体系建设.md`**

```markdown
---
status: 进行中
type: mega-project
created: 2025-01-01
---

# 健康生活体系建设

## 🎯 总体目标
在2025年建立全面的健康生活方式，包括饮食、运动、睡眠、心理健康四个维度

## 📋 子规划

- [[2025年减脂计划]] - 3个月减重5kg
- [[力量训练入门计划]] - 6个月达到基础力量水平
- [[睡眠质量改善计划]] - 建立规律的睡眠习惯
- [[正念冥想练习]] - 每天10分钟冥想

## 📊 整体进度
- 子规划总数：4
- 已完成：0
- 进行中：2
- 规划中：2
```

**子笔记会自动反链**：
- 当你在"减脂计划"中更新进度时，主笔记会自动显示
- 在 `00-系统/项目索引.md` 中可以聚合显示所有子规划

## 🔄 任务的生命周期管理

### 状态流转图

```
┌─────────┐
│  灵感库  │  ← 收集阶段
└────┬────┘
     │ 成型后
     ↓
┌─────────┐
│  规划   │  ← 拆解任务组
└────┬────┘
     │ 准备开始
     ↓
┌─────────┐
│ 计划    │  ← 设置排期
└────┬────┘
     │ 到期开始
     ↓
┌─────────┐
│ 进行中  │  ← 每日执行
└────┬────┘
     │
     ├─→ 完成任务 → [[已完成]]
     │
     ├─→ 遇到问题 → [[已暂停]]（EEDM诊断）
     │                  ↓
     │              修改计划 / 删除计划
     │                  ↓
     │              回到 [[进行中]] 或 [[规划]]
     │
     └─→ 重新评估 → 回到 [[规划]]
```

## 🎨 任务样式系统（Tasks 插件语法）

### 基础语法

```markdown
- [ ] 这是一个未完成的任务
- [x] 这是一个已完成的任务
```

### 优先级标记

```markdown
- [ ] 高优先级任务 ⏫（或 🔴）
- [ ] 中优先级任务（默认，可省略）
- [ ] 低优先级任务 ⏬（或 🔵）
```

### 日期标记

```markdown
- [ ] 有截止日期的任务 📅 2025-01-01
- [ ] 有计划开始日期的任务 ⏳ 2025-01-01
- [ ] 有 scheduled 日期的任务 🗓️ 2025-01-01
```

### 循环任务

```markdown
- [ ] 每周复盘任务 🔁 every week
- [ ] 每月支付房租 🔁 every month
- [ ] 每天运动30分钟 🔁 every day
```

### 组合使用

```markdown
- [ ] 完成减脂计划的第一次周复盘 ⏫ 📅 2025-01-08
- [ ] 购买跑步鞋 ⏬ 📅 2025-01-05
- [ ] 每周称重 🔁 every week on Monday
```

## 🚀 实战演练：创建你的第一个完整项目

### 任务：创建一个"Python学习计划"

**步骤1**：在 `01 个人/` 文件夹下创建笔记

**步骤2**：填写属性

```yaml
---
status: 计划
type: project
priority: high
created: 2025-01-01
deadline: 2025-06-30
estimated_tasks: 12
estimated_hours: 100
---
```

**步骤3**：拆解任务组

```markdown
# Python学习计划

## 🎯 学习目标
在6个月内掌握Python基础，能独立完成数据分析和自动化脚本

## 📋 任务组

### 基础语法学习（第1-2月）
- [ ] 完成Python基础教程第1-5章 📅 2025-01-15 ⏫
- [ ] 完成100道基础练习题 📅 2025-02-01
- [ ] 编写第一个CLI小工具 📅 2025-02-15

### 数据分析入门（第3-4月）
- [ ] 学习Pandas库基础 📅 2025-03-01
- [ ] 完成一个数据分析项目 📅 2025-03-31
- [ ] 学习Matplotlib可视化 📅 2025-04-15

### 实战项目（第5-6月）
- [ ] 确定毕业设计项目 📅 2025-05-01 ⏫
- [ ] 完成项目开发 📅 2025-06-15
- [ ] 编写项目文档 📅 2025-06-25

### 持续练习
- [ ] 每天编码30分钟 🔁 every day
- [ ] 每周总结一次学习笔记 🔁 every week
```

**步骤4**：将 `status` 改为 `进行中`

**步骤5**：在 `00-系统/项目索引.md` 中添加引用

```markdown
## 个人成长项目
- [[Python学习计划]] - 截止日期：2025-06-30
```

## 🎯 核心设计原则总结

1. **按生活领域分类**，而非按紧急重要分类
2. **大目标建笔记**，小任务直接记录
3. **任务组拆解**，确保可执行性
4. **属性标准化**，便于后续查询和统计
5. **双链关联**，处理超大规划
