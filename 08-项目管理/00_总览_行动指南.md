## 🎯 系统核心设计

这套系统的核心在于 **"分类"** 和 **"层级"**：

### 1️⃣ 四大领域分类（关系系统）

不用传统的"紧急重要"矩阵，而是按照 **生活关系** 来分类：

```
📁 灵感库       - 存放未成型的想法
📁 01 个人      - 健康、博学、兴趣
📁 02 家庭      - 装修、旅行、子女
📁 03 工作      - 职业任务
📁 04 事业      - 副业或创业
```

### 2️⃣ 两类任务区分

- **日常琐事**（如取快递、买牛奶）
  - ❌ 不要单独建笔记
  - ✅ 用 QuickAdd 快速录入
  - ✅ 写在日记或专门的"日常任务清单"中

- **大目标/计划**（如减肥、考证）
  - ✅ 必须新建笔记
  - ✅ 用属性（Properties）标记状态
  - ✅ 拆解为"任务组"

### 3️⃣ 改良版轻重缓急矩阵

```
┌─────────────┬──────────────┬──────────────┐
│             │   High优先级  │  普通优先级   │
├─────────────┼──────────────┼──────────────┤
│ 本周且截止是 │   紧急重要   │   紧急不重要  │
│   今天      │   （优先做）  │  （快速处理） │
├─────────────┼──────────────┼──────────────┤
│ 本周但截止是 │   不急重要   │  不急不重要   │
│ 明天以后    │  （安排时间） │  （最后处理） │
└─────────────┴──────────────┴──────────────┘
```

## 🚀 三阶段实施路线

### 第一阶段：基础环境搭建（1-2天）

**目标**：安装必要的插件，熟悉基础操作

1. ✅ 下载并安装 Obsidian
2. ✅ 安装核心插件：
   - **Tasks** - 任务管理核心
   - **Dataview** - 看板与统计
   - **Templater** - 模板快速插入
   - **QuickAdd** - 快速任务录入（推荐）

**操作指南** → [01_环境搭建指南.md](01_环境搭建指南.md)

---

### 第二阶段：构建核心框架（3-5天）

**目标**：建立文件夹结构，理解分类逻辑

1. ✅ 创建四大领域文件夹
2. ✅ 区分"日常任务"与"项目计划"
3. ✅ 掌握属性（YAML）的使用
4. ✅ 理解"任务组"的拆解逻辑

**设计说明** → [02_核心框架设计.md](02_核心框架设计.md)

---

### 第三阶段：跑通任务全流程（持续使用）

**目标**：建立日常使用习惯

1. ✅ **捕获与拆解** - 大目标变成任务组
2. ✅ **排期** - 给任务设置日期（精确到"天"）
3. ✅ **执行** - 每天只看"轻重缓急看板"
4. ✅ **复盘** - 每周检查延期任务，调整计划

**流程详解** → [03_任务全流程管理.md](03_任务全流程管理.md)

**代码模板** → [04_Dataview代码库.md](04_Dataview代码库.md)

## 💡 给初学者的一句话建议

**不要试图一次性把所有代码都写出来！**

1. **先跑通流程** → 建好4个文件夹，学会用 Tasks 插件创建带日期的任务
2. **抄作业** → 从 [04_Dataview代码库.md](04_Dataview代码库.md) 复制代码，不要自己写
3. **手动模拟** → 在没有配置好自动看板前，先手动每天打开笔记勾选任务

## 🎬 第一步行动清单

**今天就做这3件事**：

- [ ] 1. 打开 Obsidian，在工作目录下创建"个人、家庭、工作、事业"四个文件夹
- [ ] 2. 安装 `Tasks` 插件（在社区插件市场搜索"Tasks"）
- [ ] 3. 创建一个测试笔记，例如"2025年减脂计划"，写下第一条任务：
  ```
  - [ ] 制定第一周的饮食计划 ⏬ 📅 2025-01-01
  ```
